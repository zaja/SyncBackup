üìñ Introduction

SyncBackup is an application for automatic backup and folder synchronization. It supports two types of jobs, automatic scheduling, SQLite database and system tray functionality.

Key Features:
  • Dashboard with statistics overview
  • Simple and Incremental jobs
  • Automatic execution scheduling
  • ZIP backup compression
  • File filtering (exclude patterns)
  • Desktop notifications
  • System tray support

üí° Why SyncBackup?

SyncBackup was born from personal necessity after losing days or weeks of work on projects several times. The most dramatic case was when I worked on a WordPress plugin for over a week, and while testing the uninstall function, I accidentally deleted the entire project folder. At that moment, it became crystal clear - this must never happen again.

I looked for existing backup and sync solutions, but:

  • Most were too heavy and bloated for my needs
  • They were slow and consumed too many resources
  • Often not intuitive to use
  • Not flexible enough for different backup scenarios

The solution? Build my own application that is:

  • ‚úÖ Fast and lightweight - no unnecessary features
  • ‚úÖ Simple - intuitive interface
  • ‚úÖ Flexible - Simple and Incremental backups
  • ‚úÖ Automated - set it and forget it
  • ‚úÖ Reliable - never lose projects again

If you're developing software, working on important documents, or simply don't want to experience that sinking feeling when you realize hours or days of work have vanished - this application is for you.

üöÄ Quick Start

Installation and Running

  1. Install required modules: pip install -r app/requirements.txt
  2. Run the application: double-click main.pyw or python main.py

Creating Your First Job

  1. Click "‚ûï New Job"
  2. Enter job name
  3. Select type (Simple or Incremental)
  4. Set source and destination folders
  5. Configure schedule
  6. Check "Active" for automatic execution

üìÅ Job Types

Simple Job

How it works: Creates a complete copy of the source folder with date and time in the name.

Example: Documents_20251002_143000

Additional options:

  • ZIP Compression: Compress backups as ZIP files for space saving
  • Exclude Patterns: Exclude specific files/folders (e.g., .git, node_modules, __pycache__)
  • Desktop Notifications: Show notifications when job completes (success/error)

Incremental Job

How it works: Works with two concepts:

  • Sync Directory: Up-to-date working copy (destination/folder_name/)
  • Snapshots: Periodic archive copies (destination/folder_INCREMENTAL_timestamp/)

Options:

  • Preserve deleted files: Keep files in sync directory even when deleted from source
  • Create snapshots: Enable/disable periodic snapshot creation
  • Snapshot interval: How often to create snapshots (hours/days/weeks)
  • Exclude Patterns: Exclude specific files/folders (e.g., .git, node_modules, __pycache__)
  • Desktop Notifications: Show notifications when job completes (success/error)
Important: In the Backup Files tab you will only see snapshots, not the sync directory. This is by design - the sync directory is a working copy, while snapshots are archive backups for recovery.

üîΩ System Tray

The application can run in the background. Minimizing sends the application to system tray, X button closes the application.

  • Minimize: Sends to tray
  • X button: Asks for confirmation before closing
  • Right-click tray icon: Show/Quit options

‚öôÔ∏è Additional Features

Exclude Patterns

Exclude specific files/folders from backup. Examples:

  • .git,node_modules,__pycache__
  • *.tmp,*.log,Thumbs.db
  • cache/*,temp/*

ZIP Compression

Available only for Simple jobs. Compresses backup as ZIP file for space saving.

Desktop Notifications

Shows notifications when job completes:

  • Success: Green notification with file count
  • Error: Red notification with error description
  • Skipped: Yellow notification (no changes)

File Retention Policy

Automatic deletion of old backups:

  • Keep Count: Keep last N backups
  • Keep Days: Keep backups for N days
  • Keep Size: Keep up to N MB

üóÉÔ∏è Database

The application uses SQLite database (app/sync_backup.db) for:

  • Job configurations
  • Hash values for change detection
  • Execution logs
  • Backup file tracking
  • Retention policy rules

üõ†Ô∏è Troubleshooting

Job Not Running

  • Check if job is active
  • Check "Next Run" column
  • Look at logs for errors

Empty Backup

  • Check if source folder exists
  • Check read/write permissions
  • Check exclude patterns

Application Won't Start

  • Check if all modules are installed
  • Run from command prompt to see errors
  • Check Python version (3.7+)
