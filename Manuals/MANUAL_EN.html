üí° Why SyncBackup?

SyncBackup was born from personal necessity after losing days or weeks of work on projects several times. The most dramatic case was when I worked on a WordPress plugin for over a week, and while testing the uninstall function, I accidentally deleted the entire project folder. At that moment, it became crystal clear - this must never happen again.

I looked for existing backup and sync solutions, but:

  • Most were too heavy and bloated for my needs
  • They were slow and consumed too many resources
  • Often not intuitive to use
  • Not flexible enough for different backup scenarios

The solution? Build my own application that is:

  • ‚úÖ Fast and lightweight - no unnecessary features
  • ‚úÖ Simple - intuitive interface
  • ‚úÖ Flexible - Simple and Incremental backups
  • ‚úÖ Automated - set it and forget it
  • ‚úÖ Reliable - never lose projects again

If you're developing software, working on important documents, or simply don't want to experience that sinking feeling when you realize hours or days of work have vanished - this application is for you.

üìÅ Job Types

Simple Job

How it works: Creates a complete copy of the source folder with date and time in the name.

Example: Documents_20251002_143000

Additional options:

  • ZIP Compression: Compress backups as ZIP files for space saving
  • Exclude Patterns: Exclude specific files/folders (e.g., .git, node_modules, __pycache__)
  • Desktop Notifications: Show notifications when job completes (success/error)

Incremental Job

How it works: Works with two concepts:

  • Sync Directory: Up-to-date working copy (destination/folder_name/)
  • Snapshots: Periodic archive copies (destination/folder_INCREMENTAL_timestamp/)

Options:

  • Preserve deleted files: Keep files in sync directory even when deleted from source
  • Create snapshots: Enable/disable periodic snapshot creation
  • Snapshot interval: How often to create snapshots (hours/days/weeks)
  • Exclude Patterns: Exclude specific files/folders (e.g., .git, node_modules, __pycache__)
  • Desktop Notifications: Show notifications when job completes (success/error)
Important: In the Backup Files tab you will only see snapshots, not the sync directory. This is by design - the sync directory is a working copy, while snapshots are archive backups for recovery.

‚öôÔ∏è Additional Features

Exclude Patterns

Exclude specific files/folders from backup. Examples:

  • .git,node_modules,__pycache__
  • *.tmp,*.log,Thumbs.db
  • cache/*,temp/*

ZIP Compression

Available only for Simple jobs. Compresses backup as ZIP file for space saving.

Desktop Notifications (NEW in v1.3!)

Three notification modes to prevent notification spam:

  • Batch Mode (Default): Groups multiple notifications and shows a summary every 5 minutes
    • Example: "üìä Backup Summary (15 jobs): ‚úÖ 12 completed, ‚ùå 2 failed, ‚è∏Ô∏è 1 skipped"
    • Prevents 20+ notifications popping up when you return to your computer
    • Configurable interval (60-3600 seconds)
  • Immediate Mode: Shows notification for each backup job as it completes
    • Success: Green notification with file count
    • Error: Red notification with error description
    • Skipped: Yellow notification (no changes)
  • Disabled Mode: No notifications at all

Configure in Settings tab: Choose your preferred mode and batch interval.

File Retention Policy

Automatic deletion of old backups:

  • Keep Count: Keep last N backups
  • Keep Days: Keep backups for N days
  • Keep Size: Keep up to N MB

‚öôÔ∏è Settings Tab (NEW in v1.3!)

Centralized configuration for all application preferences. The Settings tab is scrollable and contains three main sections:

1. Language / Jezik

Select your preferred language:

  • Hrvatski (Croatian)
  • English

Note: Language change takes effect after application restart.

Add Your Own Language: Users can add custom languages by creating JSON files in app/languages/ directory. The application automatically scans for available languages!

2. Notification Settings

Notification Mode:

  • Immediate: Show notification for each backup job
  • Batch (Recommended): Group notifications and show summary
  • Disabled: No notifications

Batch Notification Interval: Configure how often to show batch summaries (60-3600 seconds, default: 300s = 5 minutes)

3. Windows Service Settings (Windows only)

Run as Windows Service: Enable background operation without user login

Features:

  • Auto-start with Windows
  • Runs in background 24/7
  • Perfect for servers and NAS devices

Requirements:

  • Windows operating system
  • Administrator privileges
  • pywin32 package: pip install pywin32

Service Management Buttons:

  • Install Service: Install SyncBackup as Windows Service
  • Uninstall Service: Remove the service
  • Service Status: Check current service status
Important: Installing or uninstalling Windows Service requires administrator privileges. Right-click the application and select "Run as administrator".

Adding Custom Languages

How to add a new language:

  1. Navigate to app/languages/ directory
  2. Copy en.json or hr.json
  3. Rename to your language code (e.g., de.json for German)
  4. Edit the file:
    • Change language_name to your language name
    • Change language_code to your language code
    • Translate all text strings
  5. Restart the application
  6. Your language will appear in Settings tab automatically!
